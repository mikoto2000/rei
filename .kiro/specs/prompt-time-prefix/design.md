# 設計: プロンプト時刻表示

## 概要
対話入力プロンプトの先頭に、現在時刻を `HH:mm`（24時間表記）で表示する。
既存のモデル名表示は維持し、見た目のみを変更する。

## スコープ
- 対象: 対話入力プロンプトのレンダリング処理
- 非対象: 入力確定処理、コマンド実行、履歴管理、複数行入力ロジック、ペーストモード内部ロジック

## 表示仕様
- 形式: `<HH:mm> <既存のモデル名表示>`
- 例: `09:42 gpt-5.4 > `
- 時刻はローカル時刻（現在の実行環境のタイムゾーン）を使用
- 文字列フォーマットは Java `DateTimeFormatter.ofPattern("HH:mm")`

## 実装方針
1. 既存のプロンプト文字列生成箇所を特定する。
2. 既存のモデル名プレフィックスの直前に、`LocalTime.now()` を `HH:mm` 形式で追加する。
3. 既存のプロンプト装飾（色付け・記号・末尾スペース等）を壊さない。
4. プロンプト表示以外のロジックには手を入れない。

## 変更対象（想定）
- プロンプト生成に関わるクラス（`ReiApplication` または関連レンダラ）
- プロンプト表示テスト（既存の入力レンダリング系テスト）

## テスト設計

### 単体テスト
- プロンプト文字列に `HH:mm` 形式の時刻が含まれること
- 時刻の後ろに既存モデル名が続くこと
- 既存の複数行入力・ペーストモード関連テストが回帰しないこと

### テスト戦略
- 可能なら時刻取得を分離してテスト容易性を高める（例: 時刻供給メソッドを切り出し）
- 現行設計を大きく崩さず、最小変更で検証する

## 互換性・リスク
- リスク: プロンプト固定文字列を前提にしたテストが壊れる
- 対策: 時刻部分を正規表現で検証し、固定値比較を避ける

## ロールアウト
- 変更後、入力レンダリング系テストを実行
- 可能なら全体テストを実行して非機能回帰を確認
