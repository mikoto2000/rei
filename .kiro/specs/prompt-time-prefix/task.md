# タスク: プロンプト時刻表示（t_wada TDD）

## 進め方
- 1つの振る舞いごとに **Red -> Green -> Refactor** を回す
- 先にテストで仕様を固定し、最小実装で通す
- 1サイクルごとに安全に整理して次へ進む

---

- [ ] 0. 準備
  - [ ] 0.1 プロンプト生成の実装箇所を特定する
  - [ ] 0.2 プロンプト表示に関する既存テストを特定する

- [ ] 1. サイクル1: プロンプト先頭に時刻を表示する
  - [ ] 1.1 **Red**: プロンプト先頭に時刻が無いことを確認する失敗テストを追加
  - [ ] 1.2 **Green**: 先頭に現在時刻を追加してテストを通す
  - [ ] 1.3 **Refactor**: 既存表示ロジックを崩さずに可読性を整える

- [ ] 2. サイクル2: 時刻フォーマットを `HH:mm` に固定する
  - [ ] 2.1 **Red**: `HH:mm` 以外を受け入れないテストを追加
  - [ ] 2.2 **Green**: `DateTimeFormatter.ofPattern("HH:mm")` でフォーマットしテストを通す
  - [ ] 2.3 **Refactor**: フォーマット処理をメソッド化し重複を除去する

- [ ] 3. サイクル3: 時刻の後ろに既存モデル名表示を維持する
  - [ ] 3.1 **Red**: モデル名表示が失われる失敗テストを追加
  - [ ] 3.2 **Green**: 時刻 + 既存モデル名の順序を満たすよう修正
  - [ ] 3.3 **Refactor**: 文字列組み立てを整理し、意図が読み取れる形にする

- [ ] 4. サイクル4: 既存入力機能の回帰防止（複数行入力・ペースト）
  - [ ] 4.1 **Red**: 既存テストが壊れることを確認（または不足ケースを追加）
  - [ ] 4.2 **Green**: 入力系挙動を変えずに全テストを通す
  - [ ] 4.3 **Refactor**: プロンプト表示変更が入力処理へ波及しない構造に整理

- [ ] 5. サイクル5: テストの安定化
  - [ ] 5.1 **Red**: 時刻依存で不安定なテストを再現
  - [ ] 5.2 **Green**: 正規表現検証や時刻供給の分離で安定化
  - [ ] 5.3 **Refactor**: テストコードの重複排除と命名改善

- [ ] 6. 最終確認
  - [ ] 6.1 変更対象テストを実行して全件通過を確認
  - [ ] 6.2 可能なら全体テストを実行して回帰がないことを確認
  - [ ] 6.3 requirements / design / task の整合性を最終確認
