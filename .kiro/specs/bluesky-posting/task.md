# 実装計画（t_wada式TDD）: Bluesky 投稿機能

## 方針

- 1つの振る舞いごとに **Red -> Green -> Refactor** を回す
- 先に実装を広げず、常に「今失敗している1テスト」を通す最小実装にする
- 外部APIは最初から直接叩かず、ポート/アダプタ分離でテスト容易性を確保する
