# 設計書: Chat インラインファイル添付記法

## 概要

`chat` 入力テキスト内の `` `@file:<path>` `` を添付指示として解釈し、指定ファイル内容を最終プロンプトへ組み込む。
添付記法を文字として扱いたい場合は `` \`@file:<path>` `` を利用する。

## 設計方針

- 既存の `ChatCommand` の入力処理を拡張し、AI 呼び出し直前に添付解決を行う。
- 解析対象はバッククォート囲みの `` `@file:...` `` のみとし、曖昧な終端判定を避ける。
- 添付解決失敗は入力全体の失敗にせず、該当添付のみスキップして警告を返す。
- 既存の複数行入力・ペーストモード・通常プロンプト送信の挙動を維持する。

## コンポーネント

- **ChatCommand (既存改修)**
  - ユーザー入力文字列を組み立てた後、添付記法解決を呼び出す。
- **InlineFileAttachmentResolver (新規)**
  - 役割: 入力文字列の解析、添付候補抽出、エスケープ処理。
- **ResolvedPrompt (新規データ構造)**
  - 役割: 最終プロンプト本文、添付情報、警告情報を保持。

## 解析ルール

1. 記法
- 解釈対象: `` `@file:<path>` ``
- 非解釈: `` \`@file:<path>` ``

2. 複数指定
- 入力中に複数出現した場合、出現順で処理する。

3. パス文字列
- `<path>` はバッククォート終端までをそのまま採用する。
- 前後空白はトリムせず、ユーザー入力を優先する。

## プロンプト合成

- ベースはユーザー入力テキスト。
- 各添付成功時に、ファイル名と内容を識別可能な形式で追記する。
- 添付記法トークンは最終送信文に残す。

## 失敗時挙動

- ファイル不存在・読取不可:
  - 警告を表示し、その添付のみ無効化。
- その他添付と本文は継続処理。

## テスト観点

- `` `@file:path/to/a.txt` `` が添付解釈される。
- `` \`@file:path/to/a.txt` `` が文字列として残る。
- 複数添付が順序どおり解決される。
- 空白を含むパスが解釈される。
- 一部添付失敗時に他添付と本文処理が継続される。
- 既存の chat 送信・複数行入力・ペースト関連テストが回帰しない。

## 追加設計: `@clipboard` 添付

### 1. 解析対象
- 解釈対象: `` `@clipboard` ``
- 非解釈: `` \`@clipboard` ``
- `@file` 記法と同一入力内で混在可能にする。

### 2. 添付解決
- `InlineFileAttachmentResolver`（または同等の添付解決コンポーネント）で `@clipboard` トークンを検出する。
- 検出時に OS のクリップボードから内容（テキストまたは画像）を取得し、Spring AI の `UserMessage` 添付（media/content）として構築する。
- 既存仕様どおり、トークン文字列は送信文に残す。

### 3. エラー処理
- クリップボード取得失敗時は警告を表示し、該当添付のみ無効化する。
- 他の添付（`@file` / `@clipboard`）および本文処理は継続する。

### 4. テスト観点
- `` `@clipboard` `` が添付として解釈されること（テキスト/画像）。
- `` \`@clipboard` `` が文字列として残ること。
- `@file` と `@clipboard` の混在時に両方解釈されること。
- クリップボード取得失敗時に警告を出し、処理継続すること。
