# 要件定義: 指定URL内容読み込みツール

## 背景
ユーザーが指定した URL の本文内容を会話内で参照したい要望がある。現在は URL を直接読み込む専用ツールがなく、手動での確認が必要になっているため、URL の内容取得を自動化したい。

## 対象
- 指定 URL の内容を取得するツール機能
- 取得結果の返却フォーマット
- エラー時のハンドリング

## 要件

### 要件 1: URL 指定による内容取得
**ユーザーストーリー:** ユーザーとして、URL を指定するだけでページ内容を取得したいので、取得結果をすぐ会話で確認できるようにしてほしい。

#### 受け入れ条件
1. WHEN ユーザーが URL を指定する THEN システムはその URL に対して内容取得を実行する。
2. WHEN URL 先がテキスト系コンテンツである THEN システムは本文（または抽出テキスト）を返却する。
3. WHEN 取得処理が成功する THEN システムは成功状態と取得内容を返却する。

### 要件 2: 入力バリデーション
**ユーザーストーリー:** 開発者として、不正な入力で処理が不安定にならないようにしたいので、URL の妥当性を事前に検証してほしい。

#### 受け入れ条件
1. WHEN URL が空文字または null THEN システムは取得処理を実行せず入力エラーを返却する。
2. WHEN URL スキームが `http` または `https` 以外 THEN システムは取得処理を実行せず入力エラーを返却する。
3. WHEN URL 文字列が不正形式 THEN システムは取得処理を実行せず入力エラーを返却する。

### 要件 3: エラーハンドリング
**ユーザーストーリー:** ユーザーとして、取得に失敗した理由を把握したいので、失敗時に原因が分かるメッセージを返してほしい。

#### 受け入れ条件
1. WHEN ネットワークエラーまたはタイムアウトが発生する THEN システムは失敗状態と原因種別を返却する。
2. WHEN HTTP ステータスが 4xx/5xx を返す THEN システムは失敗状態とステータスコードを返却する。
3. WHEN 本文抽出に失敗する THEN システムは失敗状態と抽出失敗メッセージを返却する。

### 要件 4: 既存機能への非影響
**ユーザーストーリー:** 開発者として、既存の会話・通知・定期実行機能を壊したくないので、新機能追加による副作用を避けてほしい。

#### 受け入れ条件
1. THE システム SHALL 既存ツール機能の挙動を変更しない。
2. THE システム SHALL URL 読み込み機能に関するテストを追加または更新する。
3. THE システム SHALL 既存テストが継続して成功することを確認できる。
