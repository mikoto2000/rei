## AI 秘書として最短で追加すべき 3 機能

### 1. タスク管理

目的:
予定管理だけでなく、「やること」を保持して AI が整理・参照できるようにする。

実装タスク:
- [x] `task` コマンド追加
- [x] `task add` `task list` `task done` `task delete` の各サブコマンド追加
- [x] タスク情報を表す `Task` エンティティ追加
- [x] タスク永続化用の SQLite テーブル追加
- [x] タスク CRUD を行う `TaskService` 追加
- [x] 優先度、期限、状態、タグを扱えるようにする
- [x] AI ツールとして `taskList` `taskCreate` `taskComplete` を追加
- [x] 期限順・優先度順で一覧できるようにする
- [x] タスク操作のユニットテスト追加
- [x] `task` コマンドの結合テスト追加
- [x] `task list` のフィルタ機能追加

### 2. 日次ブリーフィング

目的:
その日の予定・未完了タスク・関連ドキュメントをまとめて提示し、秘書らしい先回りを可能にする。

実装タスク:
- [ ] `briefing today` コマンド追加
- [ ] `BriefingService` 追加
- [ ] Google Calendar から当日予定を取得する処理を統合
- [ ] タスク一覧から未完了・期限切れタスクを集約する処理を追加
- [ ] ベクトルストアから関連文書を引く処理を追加
- [ ] 朝の概要、注意点、次アクションを生成するプロンプトを追加
- [ ] 予定ゼロ・タスクゼロ時のフォールバック文言を追加
- [ ] AI ツールとして `dailyBriefing` を追加
- [ ] ブリーフィング生成のサービス層テスト追加
- [ ] CLI から日次ブリーフィングを確認する結合テスト追加

### 3. リマインド

目的:
ユーザーが尋ねたときだけでなく、予定や期限に先回りして通知できるようにする。

実装タスク:
- [ ] `reminder` コマンド追加
- [ ] `reminder add` `reminder list` `reminder delete` の各サブコマンド追加
- [ ] リマインド情報を表す `Reminder` エンティティ追加
- [ ] リマインド永続化用の SQLite テーブル追加
- [ ] リマインドの登録・実行判定を行う `ReminderService` 追加
- [ ] 「予定の何分前」「指定日時」の両方を扱えるようにする
- [ ] 定期実行で期限到来リマインドを検出するジョブ追加
- [ ] 通知方法としてまずは標準出力通知を実装
- [ ] AI ツールとして `reminderCreate` `reminderList` を追加
- [ ] 時刻計算まわりのテスト追加
- [ ] リマインド実行条件の結合テスト追加
